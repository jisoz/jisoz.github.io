Azure VPN Connection Flow

Designed and documented the end-to-end flow of a Point-to-Site VPN connection between client devices and Azure Virtual Network. The process includes:

  • Client Initiation: Devices connect to the Azure VPN Gateway using SSTP, OpenVPN, or IKEv2 with certificates or credentials.
  • Authentication: VPN Gateway authenticates connections via certificates, Azure AD, or user credentials.
  • Secure Tunnel: A VPN tunnel encrypts all data between client and Azure.
  • IP Assignment: Clients receive private IPs from a configured address pool, appearing as part of the Azure VNet.
  • Routing: VPN Gateway routes traffic to the VM subnet, enabling communication within the VNet.
  • Access & Response: RDP requests from the client traverse the secure tunnel to VMs and responses return securely through the same path.
This setup ensures secure, reliable, and seamless remote access to Azure-hosted virtual machines.

BackUp Solution for MSSQL Database

Designed and implemented a backup strategy for APP-4-LEGAL to ensure data integrity and enable point-in-time recovery. The strategy includes:

  • Full Backup (Sunday): Complete database backup serving as the baseline.
  • Differential Backups (Monday‚ÄìFriday): Captures all changes since the last full backup; only the last full + differential are needed for restoration.
  • Transactional Log Backups (Hourly during business hours): Records all transactions for precise point-in-time recovery.
This Full + Differential + Log approach ensures faster restores, minimal data loss, and smooth recovery for large databases.
